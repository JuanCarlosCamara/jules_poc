## 2024-05-20 - Fast Path for Multi-word Positional Arguments
**Learning:** The previous fast-path implementation checked if there was only 1 positional argument. If someone passed a multi-word argument without quotes, like `hello-jules Hello World`, it would fall back to the slower `argparse` path since `len(args)` would be `2`. Additionally, using `from typing import List, Optional` adds a ~25ms startup overhead, which can be avoided entirely in Python 3.10+ by utilizing the built-in `list` and `| None` syntax.
**Action:** The fast path for positional arguments can be optimized to process multi-word arguments by simply checking if none of the arguments start with `-`, and joining the arguments with a space, entirely avoiding the need to import `argparse`. And type hints should utilize built-in alternatives over `typing` where supported to minimize startup times.
